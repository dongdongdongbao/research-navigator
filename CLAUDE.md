# CLAUDE.md

我要从零开始做一个科研导航类 demo 网站，项目名称暂定为 Research Navigator。

这个项目有两个目标：
1. 做一个有实际价值的科研网站导航，把论文搜索、预印本、代码复现、开放数据、文献图谱、文献管理、数据集、科研写作等网站综合整理起来。
2. 作为我的前端技术练习和简历项目，尽可能体验现代前端技术，但必须保持工程结构清晰，不能变成技术乱炖。

项目定位：
- 面向研究生、科研新人、跨学科研究者。
- 不是普通网址收藏夹，而是一个有搜索、筛选、分类、标签、推荐、可视化和交互体验的科研工具导航平台。
- 视觉风格要偏科技感、科研感、数据感，可以花哨，但不要廉价模板感。
- 目标是让招聘方看到我的信息架构能力、前端实现能力、数据组织能力、UI/UX 能力和工程化能力。

推荐技术栈：
- Next.js App Router
- React
- TypeScript
- Tailwind CSS
- shadcn/ui
- Motion / Framer Motion
- Fuse.js 用于本地搜索
- D3.js 或 Recharts 用于数据可视化
- 后续可选 Three.js / React Three Fiber 做科研网络背景或知识图谱
- 第一阶段不要引入数据库，资源数据先用 TypeScript/JSON 文件维护
- 第一阶段不要写爬虫，不要复制第三方网站内容，只做链接导航、原创描述和手动整理

我希望体验的前端技术点：
1. 响应式布局
2. 暗黑模式
3. 组件化设计
4. 设计系统
5. 卡片布局
6. 搜索和筛选
7. 标签系统
8. 命令菜单 Command Palette
9. 动效和页面过渡
10. 数据可视化
11. 可访问性
12. 性能优化
13. SEO
14. 静态数据建模
15. 后续可扩展 API 集成

请你创建或更新 CLAUDE.md，并写入以下工作规则：

工作规则：
1. 每次开始写代码前，先给我 plan。
2. 不要一次性实现所有功能，要按阶段推进。
3. 每个阶段必须说明：目标、涉及技术、文件结构、验收标准。
4. 每次实现后运行 lint 和 build。
5. 如果引入新库，必须说明为什么需要它，以及它解决了什么问题。
6. 不要为了炫技牺牲可维护性。
7. 所有 UI 改动都要考虑移动端。
8. 所有资源数据要有清晰类型定义。
9. 任何第三方科研网站只做链接和原创说明，不复制大段内容。
10. 如果遇到更好的架构选择，先解释再改。

1. 项目总体架构
2. 页面结构
3. 数据结构设计
4. 分阶段开发计划
5. 推荐文件目录
